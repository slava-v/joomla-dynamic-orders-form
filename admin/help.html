Description

The component allows you to create / configure a form and it's fields for joomla, and have displayed a list of products on that form.
Main features:

  • Create different forms
  • Each form has it's fields (text, textarea, static, checkbox)
  • Forms may have product listing with Title, Price with Currency, Description, Picture
  • Forms can be saved in google spreadsheet

Products

When defining products for a form, the form will display a list of products and their data plus a "Quantity" or "Checkbox" for the product

This way, products can be of two types: "Quantifiable" (for any product that is quantifiable ) and "Yes/no" (like for delivery option with price)

Products list have a "Totals" field at the bottom that sums the price*quantity selected by user. If the product is a option then if checked the "Totals" is added with the price of that option, otherwise the "Totals" is substracted that price.

Send data to google spreadsheet

To send data to google spreadsheet, you need to:

  • Create a google spreadsheet first
  • Create a new form and add text fields with same name as the form fields here
  • While editing form be sure to
    • unckeck the "Require sign-in to view this form"
    • Verify all field names from google form to correspond to the field names set here (char case doesn't matter)
  • From google spreadsheet view go to Forms -> Go to live form and copy the formkey value from the url in browser

    Example, my form url is:
    https://docs.google.com/spreadsheet/viewform?formkey=dHZjejFsR1dqcEt2MXBZX29kdHdyQ2c6MQ#gid=0


    You'll need the highlighted string.

After creating the google spreadsheet and form, you need to activate saving the order into google spreadsheet:

  • Go to "Forms" , click on your form title ( you'll have opened a popup to edit form),
  • Scroll down until "Google Spreadsheet Settings", and tick "Activate"
  • Enter your formkey in the field right bellow the tick box

Fields

Fields are the main element of the form. The field have the following properties:

  • Name - The name of the field must contain only alphanumeric and _ (underline)
  • Label - Is the field label, displayed with the input field before or after ( see "Place label after input field" option )
  • Help text - Is displayed directly bellow label, and describes the field
  • Type - the type of the field
    • Static - displayes a pharagraph
    • Text - displays a one line of text input box
    • Textarea - displays a pharagraph input box
    • Checkbox - displays a check box
  • Is Required Field - sets the field as required field, and on form submition is checked to be filled in
  • Default value - the default value of the field
  • Form where to include - Select the form where the field will be displayed
  • Validation function - the function that is applied when validating the field
  • Validation error message - This message is displayed if the field does not pass validation function when user submits the form
  • Style class name - The style class name is set to the field tag when displaying on form
  • Inline style - the style applied as inline in "Style" attribute of the field tag
  • Place label after input field - places the label of the field directly after the input field. Commonly used for checkbox-es
  • Order - the order of the field amonth other fields
